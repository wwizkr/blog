# BlogWriter 상용 개발 계획서

## 1. 상용 목표
- 개인용 툴이 아닌 **상용 운영 가능한 Windows 데스크톱 제품**으로 출시한다.
- 핵심 가치: 멀티채널 수집 + 라벨링 + 페르소나 기반 작성 + 발행 자동화.
- 1차 출시 목표: 한국어 콘텐츠 제작팀/1인 마케터 대상.

## 2. 제품 전략
## 2.1 제품 포지셔닝
- 대상 고객:
  - 콘텐츠 마케팅 대행사
  - 소규모 커머스 운영자
  - 브랜드 인하우스 마케터
- 핵심 차별점:
  - 채널별 수집/작성/발행 워크플로우 일원화
  - 형식별(블로그/SNS/게시판) 자동 변환
  - 반자동 검수 + 자동 스케줄 발행

## 2.2 에디션 전략
- `Starter`: 단일 사용자, 기본 수집/작성/수동 발행
- `Pro`: 자동 발행, 다중 페르소나, 고급 라벨링
- `Team`: 다중 계정, 승인 워크플로우, 감사 로그/리포트

## 3. 상용 아키텍처 원칙
- 기본: 오프라인 우선 Desktop App (Pyside6 + SQLite)
- 선택: 클라우드 연동 옵션(백업/팀 협업) 모듈화
- 확장성:
  - 수집기/발행기 플러그인 구조
  - LLM 벤더 추상화 (OpenAI/Gemini/추가 모델)
  - 정책 엔진(금칙어/품질/컴플라이언스) 분리

## 4. 필수 상용 요구사항
## 4.1 보안
- API 키/토큰 암호화 저장(OS 키체인 또는 암호화 Vault 파일)
- 로컬 데이터 암호화 옵션(민감 고객용)
- 자동 업데이트 시 코드서명 검증

## 4.2 안정성
- 크래시 리포트 수집(옵트인)
- 작업 큐 재시도/복구(앱 강제 종료 복원)
- 대용량 처리 시 UI 비차단(Worker 프로세스/스레드)

## 4.3 법적/정책 리스크 대응
- 수집 대상 채널 이용약관/robots 준수 정책
- 저작권/초상권 경고 및 내부 사용 정책 고지
- 생성 콘텐츠 유사도/표절 점검 옵션
- 사용자 동의 기반 로그/텔레메트리 수집

## 4.4 운영성
- 진단 리포트 내보내기(로그/환경/설정)
- 백업/복구 마법사(SQLite + 이미지 + 설정)
- 설정/데이터 마이그레이션 도구 제공

## 5. 버전 정책
- SemVer 적용
- LTS 브랜치 운영(엔터프라이즈 고객 대응)

## 6. 품질 기준(Quality Gate)
- 기능 테스트:
  - 수집 정확도/중복률
  - 라벨링 일관성
  - 형식별 생성 품질
  - 발행 성공률
- 성능 테스트:
  - 1만건 콘텐츠 인덱싱
  - 동시 작업 50개 큐 처리
- 배포 테스트:
  - Windows 10/11
  - Defender 오탐 최소화(코드서명, 빌드 위생)

## 7. 팀 구성 권장
- Product Owner 1
- Desktop Engineer 2
- Backend/Connector Engineer 1
- AI/Prompt Engineer 1
- QA 1
- DevOps/Release 0.5~1
- Designer 1 (파트타임 가능)

## 8. 단계별 상용 로드맵
## Phase A. 상용 기획 고도화 (2주)
- 고객 세그먼트 인터뷰
- 배포/지원 정책 확정
- 법무 검토 체크리스트 작성

## Phase B. MVP 제품화 (6~8주)
- 범용 수집 + 라벨링 + 형식별 생성
- 리치 에디터 + 반자동 발행
- EXE 배포 + 설치/업데이트 체계 구축

## Phase C. 베타 운영 (4주)
- 파일럿 고객 5~10개사 운영
- 장애/품질 데이터 수집
- 온보딩 문서/튜토리얼/지원 프로세스 구축

## Phase D. 정식 출시 (4주)
- 자동 발행 스케줄러 안정화
- 릴리즈/업데이트 체계 정착

## Phase E. 확장 (지속)
- 채널 커넥터 추가
- 팀 협업 기능(승인/권한/감사로그)
- 기업용 배포(정책관리/프록시/오프라인 설치)

## 9. KPI(출시 후 6개월)
- 월간 활성 사용자(MAU) 증가율: 15%+
- 기능 재사용률(주요 기능 3개 이상 사용 비율): 60%+
- 발행 성공률: 95%+
- CS 1차 응답시간: 4시간 이내
- 크래시율: 세션 기준 1% 이하

## 10. 리스크 및 대응
- 채널 API/정책 변화:
  - 커넥터 독립 배포
  - 핫픽스 채널 분리
- 생성품질 편차:
  - 카테고리별 프롬프트 템플릿
  - 품질검사 룰 + 인간검수 루프
- 법적 이슈:
  - 사용 가이드/면책/정책 명문화
  - 고위험 기능(자동 재게시) 기본 비활성화

## 11. 즉시 실행 항목(2주 스프린트)
1. 상용 요구사항 백로그 분해(보안/법적/운영)
2. 설치/업데이트 체계 PoC(자동 업데이트 포함)
3. Desktop UI 프로토타입(90% 화면 레이아웃 + 에디터 포함)
4. 발행 워크플로우 PoC(반자동 승인 -> 게시)
5. 베타 고객 인터뷰 스크립트 및 모집
