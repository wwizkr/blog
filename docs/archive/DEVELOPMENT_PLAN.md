# BlogWriter 개발 계획서

## 1. 목표
- 여행 리뷰 전용이 아닌 범용 콘텐츠 수집/생성/발행 도구를 구축한다.
- 사용자가 카테고리/키워드/수집 채널을 직접 설정할 수 있어야 한다.
- 라벨링 + 페르소나 기반 글 생성 + 발행(자동/반자동)까지 하나의 데스크톱 앱에서 처리한다.
- Windows EXE 배포를 기본으로 한다.

## 2. 요구사항 정리
1. 카테고리 + 키워드 기반 멀티 채널 크롤링 (네이버 블로그, 티스토리, 추가 SNS 채널 확장 가능 구조)
2. 본문/이미지 라벨링 저장
3. 페르소나 기반 글 생성 (블로그형, SNS형, 게시판형)
4. 자동 발행 + 반자동 발행 프로세스
5. 웹서버 없이 동작하는 EXE 우선 (필요 시 내장 로컬 서버 옵션 검토)
6. 글쓰기 에디터 필수
7. UI는 `WwizMshopCollector` 스타일 참고, 화면 90% 수준 대형 레이아웃

## 3. 제품 형태 결정
### 기본안 (권장)
- **Desktop App 단일 실행형**
- Python + Qt(Pyside6/PyQt6) 기반 GUI
- 내장 DB(SQLite 기본, 필요 시 MySQL 플러그인)
- 백그라운드 작업 스레드/큐 내장
- PyInstaller로 EXE 빌드

### 대안
- 내장 로컬 웹뷰 방식(Desktop shell + local server)을 선택 가능하게 설계
- 단, 배포/운영 복잡도 증가로 1차 릴리즈에서는 제외 권장

## 4. 아키텍처 설계
## 4.1 레이어
- `Core`: 도메인 모델, 규칙, 설정
- `Collector`: 크롤러 어댑터(네이버/티스토리/확장)
- `Labeling`: 텍스트/이미지 라벨링 파이프라인
- `Writer`: 페르소나 + 템플릿 + LLM 연동
- `Publisher`: 채널별 발행 커넥터
- `UI`: 대시보드, 수집 관리, 라벨링, 에디터, 발행 관리
- `Storage`: SQLite + 파일스토리지(이미지)

## 5. 기능 설계
## 5.1 수집
- 카테고리 템플릿: 예) IT, 뷰티, 반려동물, 금융, 여행, 육아
- 키워드 세트: 수동 등록 + 연관키워드 확장 옵션
- 채널 선택: 네이버/티스토리/확장 채널 on/off
- 결과 저장: 원문, 메타데이터, 이미지 목록, 수집 시간, 출처 URL

## 5.2 라벨링
- 텍스트 라벨: 톤, 감성, 관점, 핵심 주제, 광고성 의심 등
- 이미지 라벨: 카테고리, 품질, 분위기, 썸네일 후보
- 라벨링 엔진: Rule 기반 + AI 보조(옵션)
- 라벨링 이력 저장: 방식(rule/ai), 신뢰도, 수정 이력

## 5.3 글 생성
- 페르소나 관리: 이름, 말투, 전문성, 금지 표현, 선호 구조
- 출력 형식:
  - 블로그형(긴 글, 소제목, 이미지 본문 삽입)
  - SNS형(짧은 문장, 해시태그)
  - 게시판형(핵심 요약, 가독성 중심)
- Prompt 전략:
  - 카테고리별 가이드
  - 라벨 기반 강조점 반영
  - 출처/근거 데이터 링크 추적

## 5.4 발행
- 반자동 발행:
  - 에디터 검수 -> 승인 -> 발행
  - 채널별 최종 미리보기/수정
- 자동 발행:
  - 스케줄러 + 조건 기반(품질 점수, 금칙어 검사, 이미지 유효성)
  - 실패 시 재시도/대기열/오류 리포트
- 감사 로그:
  - 누가/언제/어디에/어떤 본문으로 발행했는지 추적

## 5.5 에디터
- 필수 요구사항:
  - Rich Text + Markdown 전환
  - 이미지 드래그/삽입/정렬
  - 템플릿 블록(소개, 요약, CTA)
  - 버전 히스토리(초안 복원)
  - 발행 전 품질 체크(금칙어/문장 길이/링크 누락)

## 6. UI/UX 설계 방향
- 기준: 기존 `WwizMshopCollector` 스타일의 업무형 대시보드
- 해상도 정책: 앱 시작 시 화면의 약 90% 크기로 실행
- 주요 화면:
  - 대시보드(수집/라벨링/생성/발행 현황)
  - 키워드/카테고리 관리
  - 수집 데이터 브라우저(원문+이미지)
  - 라벨링 검수 화면
  - 글 생성/에디터 화면
  - 발행센터(반자동/자동/스케줄/로그)

## 8. 기술 스택 제안
- Language: Python 3.12+
- GUI: Pyside6(권장) 또는 PyQt6
- DB: SQLite (기본), SQLAlchemy ORM
- Crawling: requests + bs4 + selenium(optional)
- AI: OpenAI/Gemini 추상화 레이어
- Editor: Qt WebEngine 기반 Rich Editor 또는 QScintilla/Markdown hybrid
- Packaging: PyInstaller
- Logging: structlog 또는 표준 logging + 파일 롤링

## 9. 개발 단계 로드맵
## Phase 0. 기획 고정 (1주)
- 요구사항 상세 명세
- 채널/발행 범위 확정
- UI 와이어프레임

## Phase 1. 코어/데이터 기반 구축 (2주)
- 프로젝트 스캐폴딩
- DB 스키마 + 마이그레이션 체계
- 설정/암호키 저장 체계

## Phase 2. 수집/저장 파이프라인 (2~3주)
- 카테고리/키워드 관리
- 네이버/티스토리 수집 어댑터
- 이미지 저장 및 중복 제거

## Phase 3. 라벨링 엔진 (2주)
- 텍스트/이미지 rule 라벨링
- AI 라벨링 옵션 연동
- 라벨 검수 UI

## Phase 4. 글 생성 + 에디터 (2~3주)
- 페르소나 관리
- 형식별 생성기(블로그/SNS/게시판)
- 에디터/미리보기/버전 기능

## Phase 5. 발행 프로세스 (2주)
- 반자동 승인 흐름
- 자동 발행 스케줄/정책
- 실패 처리/재시도/로그

## Phase 6. EXE 배포/안정화 (1~2주)
- PyInstaller 빌드 파이프라인
- 설정/DB 백업 복구
- 성능/에러 핸들링/릴리즈 문서

## 10. 리스크 및 대응
- 채널 정책 변경/차단: 어댑터 분리 + 백오프/재시도/대체 수집 경로
- AI 비용/품질 편차: 캐시 + 샘플링 + 룰베이스 fallback
- 발행 API 인증 복잡도: 채널별 인증 모듈 독립
