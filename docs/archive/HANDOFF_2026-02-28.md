# BlogWriter 작업 기록 (2026-02-28)

## 1) 오늘 완료한 작업
- 웹 수집 데이터 화면 개편
  - 좌측 수집 목록 / 우측 상세(2:8 레이아웃)
  - 수집 본문 상세를 Mublo Editor로 확인/수정 가능하도록 연결
  - 본문/이미지 라벨 수동 저장 동작 점검

- 에디터 적용 문제 수정
  - 수집데이터/글생성 화면 모두 Mublo Editor 초기화 경로 정리
  - HTML 코드가 텍스트로 보이던 문제 수정

- 수집 데이터 초기화 기능 수행
  - 키워드 유지, 수집글(raw_contents)/이미지(raw_images)/수집이력(crawl_jobs) 반복 초기화 수행

- 네이버/티스토리 수집 파싱 보강
  - 본문 영역 선택자 및 불필요 블록 제거(drop selectors) 정리
  - 네이버 검색 결과 링크 추출 보강(리다이렉트/이스케이프 URL 대응)

- 네이버 이미지 수집/표시 문제 수정
  - 썸네일/플레이스홀더(blank.gif 등) 필터 보강
  - 본문 HTML 이미지 URL 정규화 로직 개선
  - 네이버 mblogthumb 타입 안정화: `type=w966` 고정
    - 사유: `w1300/w1600` 일부 URL이 404 반환하여 다운로드 실패 발생
  - 기존 저장 데이터 보정
    - 네이버 이미지 URL 일괄 교체
    - local_path 누락 이미지 재다운로드 복구

- 수집 이미지 미리보기 개선
  - 로컬 파일 직접 경로 대신 서버 API로 스트리밍 제공
  - `/api/collected/images/{image_id}/file` 추가
  - 미리보기는 `local_url` 우선 사용

## 2) 확인된 원인/교훈
- 네이버 이미지에서 "크게 만들기"를 무조건 우선하면 오히려 404가 늘어날 수 있음.
- 현재 네이버는 안정 타입(`w966`)이 다운로드 성공률과 화면 표시 품질의 균형점.
- 수집 화면에서 본문이 HTML인 경우 텍스트 변환을 거치면 이미지가 사라져 보일 수 있음.
