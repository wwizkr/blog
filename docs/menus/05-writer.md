# 05. 글 작성

## 서브메뉴 (채널 우선)
1. 작성 채널관리
2. 글 작성 설정
3. 글 작성 실행
4. 페르소나 관리
5. 템플릿 관리
6. AI API 관리
7. 작성 결과/에디터

## 목적
- 등록된 채널별 설정에 따라 글을 자동/반복 작성할 수 있는 체계 제공
- 글 생성과 발행을 분리하여, 본 메뉴는 "작성" 단계만 책임

## 설계 원칙
- 채널 우선: 모든 작성 정책은 채널 단위로 독립 관리
- 자동화 중심: 채널별 주기/조건 기반 반복 작성 지원
- 설정과 실행 분리: 정책은 `글 작성 설정`, 실제 생성은 `글 작성 실행`
- 저장 일원화: 자동 작성 결과는 게시판형 단일 저장소에 누적
- 발행 분리: 발행 스케줄/발행 처리 로직은 `06. 발행` 메뉴에서 담당

## 1) 작성 채널관리 (신규 최상단)

### 화면 목적
- 글 작성 대상 채널을 등록/수정/삭제하고, 채널별 필수 정보를 관리

### 핵심 기능
- 채널 등록: 채널 타입 선택 후 인증 정보 및 작성 관련 메타 입력
- 채널 수정: 채널명, 상태, 기본 카테고리/태그 정책, 제약 정보 수정
- 채널 삭제: 소프트 삭제(이력 유지) + 자동화 중지
- 채널 상태: `활성`, `중지`, `인증오류`, `만료임박`
- 채널 검증: 연결 테스트 및 최소 필수 필드 유효성 검사

### 채널별 관리 정보(공통)
- 식별 정보: 채널명, 채널 타입, 채널 ID, 소유자
- 작성 제약: 제목/본문 길이, 허용 마크업, 링크/이미지 제한
- 기본 분류: 기본 카테고리, 기본 태그 정책, 기본 공개 범위
- 운영 정보: 생성일, 마지막 인증일, 마지막 작성 성공일

## 2) 글 작성 설정

### 화면 목적
- 상단에 등록된 채널 탭을 배치하고, 채널별 작성 정책을 설정

### UI/동작 원칙
- 상단 채널 탭: `전체(전역)` + `등록 채널별 탭`
- 채널 탭 전환 시 해당 채널 정책을 즉시 편집/저장
- 전역 기본값 + 채널 오버라이드 2계층 구조 유지

### 채널별 설정 항목
- 기본 페르소나, 기본 템플릿, 기본 AI API 프로필
- 톤/길이/독자수준/금칙어/사실성 우선도/창의성 강도
- SEO 옵션: 핵심 키워드, 제목 규칙, 메타 설명 길이
- 품질 게이트: 최소 글자수, 금지 표현, 필수 섹션
- 자동화 옵션: 작성 주기, 실행 시간대, 1회 생성 개수, 실패 재시도 정책

### 자동 작성 정책
- 각 채널은 독립 스케줄 정책으로 반복 작성 가능
- 동일 주제라도 채널별 규칙에 맞는 별도 결과 생성
- 비활성 채널은 자동화 대상에서 즉시 제외

## 3) 글 작성 실행

### 화면 목적
- 수동 실행 또는 예약 트리거 확인을 통해 글 생성 작업 수행

### 실행 플로우
1. 채널 선택(단일/복수)
2. 채널 정책 로드(탭 설정값)
3. 소스 선택(키워드/수집데이터/직접 입력)
4. 초안 생성 및 품질 게이트 검사
5. 작성 결과 저장(게시판형 단일 테이블)

## 4) 페르소나 관리
- 페르소나 생성/수정/버전 관리
- 채널별 추천 페르소나 매핑 지원

## 5) 템플릿 관리
- 템플릿 구조/슬롯/채널 제약 관리
- 채널별 템플릿 오버라이드 지원

## 6) AI API 관리
- 제공자/모델/인증키/비용 제한 관리
- 채널별 기본 모델 매핑 및 폴백 체인 제공

## 7) 작성 결과/에디터

### 핵심 원칙
- 자동 작성된 글은 게시판 형태로 조회
- 저장은 하나의 통합 테이블에서 관리
- 본 메뉴는 작성 결과 편집/검수까지만 담당
- 실제 발행은 발행 스케줄에 따라 별도 처리

### 핵심 기능
- 목록: 채널, 상태, 생성시각, 품질점수, 예약 발행 여부
- 상세: 본문 에디터, 품질 검사 패널, 버전 이력
- 후처리: 축약/확장/톤 변경/근거 강화

## 데이터 모델 초안 (작성 기준)
- `writing_channels`: 작성 채널 마스터
- `channel_writer_policies`: 채널별 작성 정책(자동화 포함)
- `writer_profiles`: 전역 기본 설정/프리셋
- `personas`: 페르소나 + 버전
- `templates`: 템플릿 + 슬롯 + 채널 제약
- `ai_providers` / `ai_models` / `ai_credentials`
- `writing_jobs`: 생성 실행 단위(수동/자동 트리거)
- `writing_posts`: 자동/수동 작성 결과 통합 저장 테이블(게시판형)

## 메뉴 수정 반영 요약
1. 등록 채널별 설정에 따라 자동/반복 글 작성 지원
2. `작성 채널관리`를 최상단 메뉴로 추가(등록/수정/삭제)
3. `글 작성 설정`에 채널 탭을 두고 채널별 정책 관리
4. 자동 작성 글은 단일 테이블(`writing_posts`)에 저장, 발행은 별도 메뉴에서 처리

## 글 작성 메뉴 수정 계획 반영
1. 등록된 채널에는 각 채널별 설정에 따라 자동화/반복 글 작성이 가능하도록 `channel_writer_policies` 기반 정책을 적용한다.
2. `작성 채널관리` 메뉴를 최상단에 배치하고, 채널 등록/수정/삭제 및 채널별 필수 정보 관리를 담당한다.
3. `글 작성 설정` 상단에 채널 탭(`전체 + 등록 채널`)을 두고, 기존 기본 작성 정책을 채널별로 동일하게 설정할 수 있도록 한다.
4. 설정에 따라 자동 작성된 글은 게시판 형태의 단일 테이블(`writing_posts`)에 저장하며, 발행은 발행 스케줄에 따라 별도 메뉴에서 처리한다.
