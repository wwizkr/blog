# 06. 발행

## 서브메뉴 (설정 우선, 채널관리 제거)
1. 발행 설정
2. 발행 실행
3. 발행 이력

## 목적
- 채널별 발행 정책(주기/모드) 기반 자동 반복 발행 운영
- 발행 채널 관리는 별도 메뉴 없이 `발행 설정`에서 채널별 정책으로 통합 관리

## 운영 원칙
- 채널별 발행 정책 저장: 각 채널별로 `발행 주기(분)`, `모드(auto/semi_auto)`, `포맷`, `API URL`
- 자동 반복 발행: `auto` 채널은 주기마다 발행 가능한 글을 자동 처리
- 즉시 실행 분리: `발행 실행`은 자동 반복 상태 확인/수동 트리거/로그 확인 역할
- 이력 가시성: `발행 이력`에서 채널/상태/메시지 기반 추적
