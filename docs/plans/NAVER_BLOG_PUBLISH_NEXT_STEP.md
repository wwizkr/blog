# Naver Blog Publish Next Step

## 왜 이 문서가 필요한가

다음 단계는 `[[IMAGE:id]]` 슬롯을 실제 발행 채널 이미지로 바꾸는 것이다.  
그중에서도 `Naver Blog`가 가장 어렵고, 이 채널의 제약을 먼저 알아야 전체 publish adapter 구조를 고정할 수 있다.

따라서 다음 작업의 최우선 과제는 `네이버 블로그 발행/이미지 업로드 방식 검증 및 설계 확정`이다.

## 다음에 바로 해야 할 일

1. 네이버 블로그 발행 방식 확정
- API 기반이 가능한지
- Selenium 기반 브라우저 자동화가 필요한지
- 이미지 업로드와 본문 작성이 어떤 흐름으로 연결되는지

2. 필요한 사용자 제공 정보 정의
- 계정
- 비밀번호 또는 앱 비밀번호/2차 인증 예외 방식
- 로그인 유지 정책
- 블로그 선택 방식

3. 세션/보안 정책 정의
- 로그인 세션 저장 위치
- 세션 만료 시 재로그인 정책
- CAPTCHA 또는 추가 인증 발생 시 수동 개입 방식

4. 이미지 업로드 정책 정의
- 로컬 이미지 파일을 어떤 단계에서 업로드할지
- 업로드 후 어떤 식별자를 받을 수 있는지
- 본문 내 이미지 위치를 어떻게 고정할지

5. 실패/복구 정책 정의
- 로그인 실패
- 이미지 일부 업로드 실패
- 글 저장 실패
- 발행 직전 실패

## 이 단계에서 구현하지 않을 것

- WordPress/Tistory adapter 구현
- 전체 채널 공통 UI 완성
- 자동 발행 스케줄 고도화

지금은 `네이버 블로그가 실제로 어떤 제약을 가지는지`를 먼저 확정하는 단계다.

## 완료 기준

- 네이버 블로그 발행을 위한 인증 방식이 확정되어야 한다.
- 로컬 이미지 업로드 흐름이 단계별로 정리되어야 한다.
- `[[IMAGE:id]]` 슬롯이 네이버 블로그 본문 이미지로 치환되는 설계가 확정되어야 한다.
- 사용자가 무엇을 제공해야 하는지 명확해야 한다.
- 이후 구현 순서를 바로 뽑아낼 수 있어야 한다.
