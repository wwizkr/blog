# TODO 이관 규칙

## 상태 정의
- `done`: 구현/검증/문서 반영까지 완료된 항목
- `deferred`: 요구사항 보류 또는 외부 의존성으로 연기된 항목
- `pending`: 아직 작업 전 또는 진행 중

## 체크 규칙
- 체크박스 `[x]`: `done`
- 체크박스 `[ ]`: `pending` 또는 `deferred`
- `deferred` 항목은 라인 끝에 `(deferred: 사유)`를 명시

## 이관 절차
1. 작업 완료 시 해당 메뉴 TODO 체크
2. 변경 파일/검증 커맨드를 작업 로그에 기록
3. 보류 시 사유와 재개 조건(의존성/일정)을 같은 줄에 기록
