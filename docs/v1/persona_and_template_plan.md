# 기본 페르소나 및 템플릿 초기 설정 계획서
Version: 1.0  
목적: 설치 직후 즉시 콘텐츠 생성이 가능하도록 기본 데이터 자동 구성

---

# 1. 개요

본 문서는 WriterService 기반 콘텐츠 생성 시스템에서  
초기 설치 시 기본 페르소나(Persona) 및 템플릿(Template)을 자동 설정하기 위한 설계 문서이다.

목표는 다음과 같다.

1. 사용자가 별도 설정 없이 즉시 글 생성 가능
2. 채널별 기본 템플릿 제공
3. 다양한 톤의 페르소나 기본 제공
4. 확장 가능하고 버전 관리 가능한 구조 유지

---

# 2. 기본 구성 전략

초기 설치 시 다음 데이터를 자동 생성한다.

- 기본 페르소나 4~6개
- 기본 템플릿 5~10개
- 기본 Writing Channel 3~4개
- 기본 AI Provider 1개 (선택)

수집 및 라벨링 시스템과 독립적으로 동작한다.

---

# 3. 기본 페르소나 설계

초기 제공 페르소나는 과도하지 않게 4~6개로 제한한다.

## 3.1 정보형 분석가

- name: 스마트 가이드
- age_group: 30대
- gender: 중립
- personality: 논리적
- speech_style: 설명형
- tone: 정보형
- style_guide: 구조화된 문장, 목록 활용
- banned_words: 과도한 감탄사

용도:
- 블로그 가이드
- 비교 분석 글
- SEO 최적화 글

---

## 3.2 친근한 리뷰어

- name: 생활 리뷰러
- age_group: 20~30대
- personality: 밝음
- speech_style: 구어체
- tone: 후기형
- style_guide: 경험 중심 서술
- banned_words: 전문용어 남용

용도:
- 제품 리뷰
- 여행 후기
- 체험기

---

## 3.3 트렌드 SNS형

- name: 트렌드 큐레이터
- age_group: 20대
- personality: 감각적
- speech_style: 짧은 문장
- tone: 감성형
- style_guide: 리듬감 있는 구성
- banned_words: 장문 설명

용도:
- SNS 요약 콘텐츠
- 짧은 정보 전달 글

---

## 3.4 커뮤니티 정보 공유형

- name: 정보 공유러
- age_group: 30~40대
- personality: 실용적
- speech_style: 담백함
- tone: 정보형
- style_guide: 핵심 위주 구성
- banned_words: 과장 표현

용도:
- 게시판 정보 공유 글
- 실용 팁 정리

---

# 4. 기본 템플릿 설계

템플릿은 구조 유형별로 제공한다.

## 4.1 블로그 가이드형

- template_type: blog
- 구조:
  - 도입
  - 핵심 요약
  - 상세 설명
  - 정리
  - CTA

---

## 4.2 리스트형 (Top N)

- template_type: blog
- 구조:
  - 도입
  - 1~N번 항목 정리
  - 비교 요약
  - 마무리

---

## 4.3 비교 분석형

- 장점/단점 구조
- 표 기반 비교
- 추천 대상 구분

---

## 4.4 SNS 요약형

- 3~5문장 핵심 요약
- 해시태그 포함 가능

---

## 4.5 커뮤니티 정보 공유형

- 핵심 정보 정리
- 주의사항
- 댓글 유도 문구

---

# 5. 기본 Writing Channel 설정

초기 설치 시 자동 생성:

| code  | display_name | channel_type |
|-------|--------------|--------------|
| blog  | 블로그       | blog         |
| sns   | SNS          | sns          |
| board | 커뮤니티     | board        |

---

# 6. 초기 데이터 삽입 방식

## 6.1 JSON 기반 Seed 방식 (권장)

설치 시 JSON 파일을 로딩하여 DB에 삽입한다.

장점:

- 수정 용이
- 다국어 확장 가능
- 버전 관리 가능
- 배포 시 유지보수 쉬움

예시 구조:

{
  "personas": [...],
  "templates": [...],
  "channels": [...]
}

---

## 6.2 중복 방지 전략

- 동일 name 또는 code 존재 시 삽입하지 않음
- template_version 관리
- 업그레이드 시 신규 버전만 추가

---

# 7. 기본 AI Provider 설정

초기 설치 시:

- provider 레코드만 생성
- API 키는 사용자 입력

설치 후 첫 실행 시 API 설정 안내 가능

---

# 8. UX 설계 전략

초기 실행 시 다음 선택 화면 제공 가능:

1. 기본 설정 자동 적용
2. 기본 페르소나 선택
3. 기본 템플릿 선택
4. 나중에 수정 가능 안내

---

# 9. 버전 관리 전략

템플릿은 version 필드 활용.

예:
- template_version = 1.0
- 업데이트 시 1.1, 2.0 추가
- 기존 사용자 설정 유지

---

# 10. 향후 확장 계획

- 라벨 기반 템플릿 자동 추천
- 성과 기반 템플릿 우선 추천
- 페르소나 자동 생성 기능
- 채널별 프롬프트 자동 최적화

---

# 11. 실행 단계

Phase 1:
- 기본 페르소나 정의
- 기본 템플릿 정의
- JSON seed 파일 작성

Phase 2:
- 설치 시 자동 삽입 로직 구현
- 중복 체크 로직 구현

Phase 3:
- UI에서 수정/비활성화 가능
- 기본값 복원 기능 추가

---

# 결론

초기 페르소나 및 템플릿 세팅은 단순 샘플 데이터가 아니라  
콘텐츠 자동 생성 엔진의 출발점이다.

잘 설계된 기본 세트는:

- 설치 직후 즉시 사용 가능
- 다양한 채널 대응 가능
- 구조적으로 확장 가능
- 유지보수와 버전 관리가 용이하다
