# 민감정보 저장/조회 정책

## 대상
- 비밀번호
- API Key / Token
- OAuth Secret

## 저장 원칙
- DB 저장 시 평문 저장 최소화
- UI 조회 시 원문 노출 금지(마스킹 표시)
- 수정 시 빈 값은 기존 값 유지, 입력 시에만 갱신

## 조회 원칙
- 목록/카드 UI에는 마스킹 값만 표시
- 상세 폼 진입 시에도 비밀번호/API 키는 placeholder 기반으로 기존 존재만 표시
- 로그/알림에 비밀값 포함 금지

## 운영 점검
- 릴리즈 전 민감정보 노출 점검 항목 필수
- 환경변수 alias 존재 여부 검증(운영/개발 분리)
