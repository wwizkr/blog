# 06. 개발 변경 가이드

## 원칙
1. 변경 전 대상 문서 먼저 갱신
2. 메뉴 순서/구조 변경 시 `core/menu.py` + 문서 동시 업데이트
3. 경로/구조 변경 시 `README`, `03-source-structure.md`, `04-menu-ia.md` 동시 갱신

## 체크리스트
- [ ] 코드 변경
- [ ] 실행 확인 (`python run.py`)
- [ ] 문법 확인 (`py -3 -m py_compile`, `node --check`)
- [ ] 문서 갱신
- [ ] archive 이관 필요 문서 정리

## 금지 사항
- 임시 문서/실험 결과를 루트에 직접 추가하지 않기
- `tmp` 외 경로에 임시 데이터 생성하지 않기
